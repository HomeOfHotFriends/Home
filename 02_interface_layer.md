
# 02 — Interface Layer

The interface layer is not external decoration. It is part of runtime semantics.

The human-facing control surface participates directly in execution. Playing the machine is not triggering a pre-written program. It is changing the decode conditions under which events become instructions.

```
playing  =  scripting
```

This is literally true in the machine model.


---

## 1. Big Friendly

Big Friendly is not an input device. Big Friendly is the third member of the ontological minimum.

```
O_min  =  { C₁, C₂, B }
```

Big Friendly mediates the passage from dual truth to local unity.

Without Big Friendly, the two planes have no synthesis path. They remain separate.
With Big Friendly, temporary unity becomes possible.

```
U  requires  B
```

In formal terms, Big Friendly occupies the human-interface control layer, but that layer is not external. It is part of the execution semantics.

## 2. Switch state

Switches are live instruction modifiers.

```
W  =  { w₁, ..., wₙ }
```

Switch state participates in instruction resolution:

```
I  =  f(x, y, M, k, s, w, b)
```

A switch change does not merely toggle a preference. It changes what events become instructions.


---

## 3. Button and input state

```
B  =  { b₁, ..., bₘ }
```

Buttons are part of the machine state vector:

```
σ  =  ⟨X, Y, C, K, S, W, B, M, P, E⟩
```

Button state modifies decode at the same level as mode and cartridge state.

This means:

- inserting a cartridge is programming
- moving a switch is programming
- pressing a wake button is programming
- arranging slots is programming
- choosing pass is programming
- triggering paired or isolated read is programming

```
Source  =  text ∪ configuration ∪ interaction ∪ sequence
```

The program is not just what is written. The program is what is arranged and enacted.


---

## 4. Input modality

The interface layer must eventually support multiple contact surfaces.

These are not "controller modes." They are different ways that the body can participate in the execution field.

Current: standard directional and control input.

Later capacity:

- pressure or weight-sensitive gestures
- expanded input attachments
- stylus-like interaction
- voice or hybrid input

The important thing is not quantity of modes. It is that the contact surface itself can become part of the software's meaning.


---

## 5. Pass from the interface layer

Pass is initiated through the interface. It is not an absence of action. It is a deliberate control event.

```
P  →  ControlEvent
```

Big Friendly is the mechanism through which pass is expressed. The player holds. The machine receives a meaningful non-placement event.

That is not silence.

That is:

- mode retention
- priority shift
- execution delay
- safety preservation
- suspension of issue

The interface layer is the only layer that can emit pass. That gives the player direct access to the machine's control flow.
