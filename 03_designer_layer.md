
# 03 — Designer Layer

The designer layer is the set of rules governing what cartridges are, what slots do, and how the machine gets arranged before it is played.

Arrangement is authorship. The way the machine is assembled is itself part of the software meaning.


---

## 1. Cartridge identity

A cartridge is not a passive file. It is a stateful machine participant.

```
k  =  ⟨id, mode, resource, dream, awake⟩
```

Cartridge states:

```
k.mode  ∈  { sleeping, awake, buried, dreaming, active }
```

State meanings:

| State | Meaning |
|-------|---------|
| `sleeping` | buried in Home; contributes to dream layer |
| `awake` | participates in shared runtime reality |
| `dreaming` | in modified/altered state under Y-isolation |
| `active` | currently executing or synthesized |
| `buried` | inserted but not yet addressed |

A sleeping cartridge contributes to a shared dream layer:

```
Sleeping(p)  →  DreamContribution(p)
```

An awake cartridge contributes to executable shared reality:

```
Awake(p)  →  RuntimeContribution(p)
```

Cartridge logic is not passive storage. It is stateful participation.


---

## 2. Slot grammar

A slot is not a storage receptacle. A slot is an operator.

```
S_n(k)  =  cartridge k placed in slot n
```

Slot position alters the meaning of the inserted cartridge:

```
Sem(S₁(k))  ≠  Sem(S₂(k))
```

Therefore:

```
Meaning(k)  =  g(k, n, Σ)
```

where `n` is slot position and `Σ` is the whole slot arrangement.

Cartridge meaning is relational, not intrinsic.

```
same cartridge  +  different slot topology  =  different executable role
```

This is one of the deepest structural laws of the machine.


---

## 3. Slot 10 (the hub slot)

Slot 10 carries special governance authority.

A cartridge in slot 10 may produce:

- machine-wide timing rules
- machine-wide routing authority
- cross-slot modulation privileges
- meta-logic

This is where the machine stops being merely modular and starts becoming governed.

The slot 10 cartridge is not simply another participant. It is the world-law cartridge.


---

## 4. Cartridge lifecycle

A cartridge can pass through the following states over time:

```
insertion  →  activation  →  combination  →  modification
          →  freezing    →  extraction   →  reinsertion
          →  retirement
```

These are not metaphors. They are distinct machine states with distinct consequences for execution.

**Freezing** is especially important. A frozen cartridge has committed its process to an object. It no longer requires its full original toolchain to function. It has become lighter.

```
live process  →  committed result  =  frozen cartridge
```

This is one of the machine's core productivity moves.


---

## 5. Cartridge Forge

The Cartridge Forge is the system that produces new stable cartridge objects from live machine states.

It extracts:

- full products
- skeletons (pattern bones, rule skeletons)
- sound bones
- world fragments
- partial reusable artifacts

The Forge is not hidden save logic. It is explicit software.

Outputs from the Forge are reinsertable artifacts, not dead files. They return to the ecosystem as first-class cartridge citizens.

```
finished work  →  new cartridge  →  reinsertable into any compatible slot
```


---

## 6. Arrangement as authorship

Because slot position is an operator, the designer's act of arranging the machine is an act of composition.

```
A₁ ≠ A₂  →  O₁ ≠ O₂
```

Different arrangements produce different ontologies. Different placements produce different meanings from the same cartridges.

This is not a metaphor. It is a structural consequence of slot grammar.

The designer is always already programming, even before a single event fires.
